# 红黑树简介

## 1. 什么是红黑树？

红黑树是一种自平衡二叉搜索树，也称为 RB 树（Red-Black Tree）。它在维护二叉搜索树性质的基础上，通过节点颜色和额外的规则保持树的平衡。

红黑树的重要特征：
- 每个节点要么是红色，要么是黑色。
- 根节点是黑色。
- 每个叶子节点（通常表示空节点 `null`）被视为黑色。
- 如果一个节点是红色，则它的两个子节点必须是黑色。
- 从任意节点到其所有后代叶子节点的简单路径上，必须包含相同数目的黑色节点。

因此，红黑树允许比 AVL 树更宽松的平衡条件，但仍能保证 `O(log n)` 的查找、插入和删除。

## 2. 红黑树的五条性质

1. 每个节点是红色或黑色。
2. 根节点是黑色。
3. 所有叶子节点（空节点）都是黑色。
4. 任何红色节点的两个子节点都是黑色。
5. 从任意节点到其所有空叶子节点的路径上，具有相同数量的黑色节点。

这五条性质共同约束了树的形态，使其在节点数增长时高度仍保持 `O(log n)`。

## 3. 数形结合：红黑树的几何意义

红黑树通过“颜色”这个额外的数值来控制树的形态。

- “黑高”表示从节点到叶子路径上黑色节点的数量。
- 节点的红黑颜色决定了是否允许子树高度差出现。

从形态上看，红黑树允许某些路径比其他路径略长，但这种不对称最多增加一倍。因此树形是“松弛平衡”的，而不是像 AVL 树那样严格对称。

## 4. 红黑树的高度界限

红黑树的黑高 `bh` 与节点总数 `n` 的关系是：

- 最短路径长度为 `bh`。
- 最长路径长度不超过 `2 * bh`。

因此，红黑树高度 `h` 有上界：`h <= 2 * log₂(n + 1)`。

这意味着查找、插入、删除仍然是 `O(log n)`。

## 5. 主要操作与修复逻辑

### 5.1 查找

红黑树的查找与普通 BST 完全相同，按节点值比较向左或向右搜索。

由于高度被平衡约束控制，查找时间为 `O(log n)`。

### 5.2 插入

插入新节点时，首先按 BST 规则插入一个红色节点。
然后通过“颜色调整”和“旋转”修复红黑性质。

常见修复情形：
- 叔叔节点为红色：重新着色，继续向上修复。
- 叔叔节点为黑色：通过旋转和重新着色完成修复。

红黑树插入后最多需要 `O(log n)` 次旋转和颜色调整。

### 5.3 删除

删除是红黑树中最复杂的操作。

基本步骤：
1. 按 BST 规则删除节点。
2. 如果删除的是黑色节点，可能破坏黑高性质。
3. 通过“借黑色”与旋转逐层修复。
4. 删除后最多需要 `O(log n)` 次旋转和颜色调整。

尽管过程复杂，但时间复杂度仍然是 `O(log n)`。

## 6. 旋转与颜色调整

红黑树主要用两种旋转：

- 左旋（Left Rotate）：将节点右子树上升，当前节点下降为左子树。
- 右旋（Right Rotate）：将节点左子树上升，当前节点下降为右子树。

配合颜色翻转，红黑树能够在局部修复不平衡后保持整体性质。

## 7. 插入修复典型情形

假设新插入的节点 `z` 为红色，且其父节点 `p` 也是红色，那么违反性质4。修复主要依赖于 `z` 的叔节点 `u` 的颜色和 `z`、`p` 的相对位置。

- 情形 1：`u` 是红色。
  - 说明 `p` 和 `u` 组成了一个红色对，祖父节点 `g` 一定是黑色。
  - 处理：将 `p` 和 `u` 涂成黑色，将 `g` 涂成红色；再把 `z` 指向 `g` 继续向上修复。
  - 结果：局部高度保持不变，顶层可能继续调整。

- 情形 2：`u` 是黑色，且 `z` 与 `p` 在同一侧（LL 或 RR）。
  - 处理：对 `g` 做一次旋转，并把 `p` 涂黑、`g` 涂红。
  - 例如 LL：右旋 `g`；RR：左旋 `g`。
  - 结果：通过一次旋转将红色节点移到更高位置，恢复性质。

- 情形 3：`u` 是黑色，且 `z` 与 `p` 在不同侧（LR 或 RL）。
  - 处理：先对 `p` 做一次旋转，再对 `g` 做一次旋转；最终把新根涂黑，把其子节点涂红。
  - 例如 LR：先左旋 `p`，再右旋 `g`。
  - 结果：把“折线”结构调整成“直线”再旋转，恢复红黑性质。

这三种情形从数形结合来看：

- 情形 1 只调整颜色，不改变形态高度；
- 情形 2 直接修正偏斜形态；
- 情形 3 先把不规则形态先局部展开，再统一旋转。

## 8. 删除修复详细流程

红黑树删除最难的部分是删除黑色节点，因为它会破坏到叶子路径上的黑色节点数量一致性。

当被删除节点替代的节点 `x` 为黑色时，`x` 的位置会产生“额外的黑色”，我们把它称为“double black”。

### 删除修复的主要情形

假设当前节点 `x` 是“double black”，其兄弟节点为 `s`。

- 情形 1：`s` 是红色。
  - 处理：将 `s` 涂黑、父节点 `p` 涂红；然后对 `p` 进行一次旋转，使 `s` 变为黑色兄弟。
  - 目标：转换为情形 2、3 或 4，便于后续处理。

- 情形 2：`s` 是黑色，且 `s` 的两个子节点都是黑色。
  - 处理：将 `s` 涂红，把 `x` 的“double black”上移到 `p`。
  - 结果：`p` 的路径失衡向上传播，继续修复。

- 情形 3：`s` 是黑色，且 `s` 的一个子节点是红色，另一个是黑色，且红色子节点在远侧。
  - 处理：对 `p` 进行旋转，使 `s` 的红色子节点成为新的兄弟根，重新着色。
  - 结果：可以直接转到情形 4。

- 情形 4：`s` 是黑色，且 `s` 的远侧子节点是红色。
  - 处理：对 `p` 旋转，`s` 变成 `p` 的父节点；将 `s` 的颜色赋给 `p`，并把 `s` 的子节点涂黑。
  - 结果：消除 `x` 的“double black”，恢复红黑性质。

### 数形结合理解删除修复

删除修复过程实际上是在“借黑色”或“重新分配黑高”：

- 情形 2 相当于把黑色高度从兄弟子树借走，继续在父节点处修复；
- 情形 4 相当于通过旋转重新分配路径长度，使失衡路径与兄弟路径重新对齐。

## 9. 红黑树的代码示意

下面给出一个简化的红黑树插入和删除辅助操作框架，用于理解核心思想。

```cpp
enum Color { RED, BLACK };

struct Node {
    int key;
    Color color;
    Node* left;
    Node* right;
    Node* parent;
    Node(int k) : key(k), color(RED), left(nullptr), right(nullptr), parent(nullptr) {}
};

class RedBlackTree {
public:
    Node* root;
    Node* nil; // 伪叶子节点，颜色为黑色

    RedBlackTree() {
        nil = new Node(0);
        nil->color = BLACK;
        nil->left = nil->right = nil->parent = nullptr;
        root = nil;
    }

    void leftRotate(Node* x) {
        Node* y = x->right;
        x->right = y->left;
        if (y->left != nil) y->left->parent = x;
        y->parent = x->parent;
        if (x->parent == nil) root = y;
        else if (x == x->parent->left) x->parent->left = y;
        else x->parent->right = y;
        y->left = x;
        x->parent = y;
    }

    void rightRotate(Node* y) {
        Node* x = y->left;
        y->left = x->right;
        if (x->right != nil) x->right->parent = y;
        x->parent = y->parent;
        if (y->parent == nil) root = x;
        else if (y == y->parent->right) y->parent->right = x;
        else y->parent->left = x;
        x->right = y;
        y->parent = x;
    }

    void insertFixup(Node* z) {
        while (z->parent->color == RED) {
            if (z->parent == z->parent->parent->left) {
                Node* y = z->parent->parent->right;
                if (y->color == RED) {
                    z->parent->color = BLACK;
                    y->color = BLACK;
                    z->parent->parent->color = RED;
                    z = z->parent->parent;
                } else {
                    if (z == z->parent->right) {
                        z = z->parent;
                        leftRotate(z);
                    }
                    z->parent->color = BLACK;
                    z->parent->parent->color = RED;
                    rightRotate(z->parent->parent);
                }
            } else {
                // 右侧对称
            }
        }
        root->color = BLACK;
    }
};
```

这段示例说明了插入修复中的“颜色重绘 + 旋转配合”结构。删除修复代码更复杂，但思路一致：通过重绘与旋转逐步消除双重黑色。

## 10. 红黑树与 2-3-4 树的关系

红黑树的一个重要理解角度是：它等价于一棵“二-三-四树”的二叉表示。

- 一个黑色节点可以视为一个 2-3-4 树节点。
- 一个红色子节点表示 2-3-4 树节点内部的键连接。

从这一角度看，红黑树的红色链接其实相当于 2-3-4 树中的“合并节点”，它把两个节点临时连接成一个更大的节点。

这一理解帮助我们把红黑树的颜色约束转化为 2-3-4 树的“节点容量约束”，从而更直观地把握其平衡机制。

## 11. 红黑树的优势与适用场景

- 平衡代价较低：插入删除只需局部修复，旋转次数有限。
- 动态性能稳定：适合读写混合场景。
- 工程应用广泛：许多库和语言运行时都基于红黑树实现有序集合。

## 12. 结语

红黑树在“数”和“形”之间建立了柔性约束：

- 颜色这类离散数值，可以表示路径上的黑高；
- 旋转这类形态变换，可以把不对称树形调回平衡。

这样，红黑树成为一种既高效又工程友好的自平衡 BST，适合实际复杂数据集合的管理。
