# B 树 与 B+ 树 简介

## 1. 什么是 B 树？

B 树是一种自平衡的多路搜索树，广泛用于数据库和文件系统的索引结构。

它的核心特点是：
- 每个节点可以包含多个关键字和多个子节点。
- 所有叶子节点都处于同一层。
- 节点的关键字按升序排列。
- 非叶子节点的关键字把子树范围划分为有序区间。

B 树通过允许节点存储多个关键字，将树的高度降低到 `O(log n)`，适合磁盘存储场景。

## 2. B 树的定义与性质

假设 B 树的阶为 `m`：

- 每个节点最多有 `m` 个子节点。
- 每个非根节点至少有 `⌈m/2⌉` 个子节点。
- 每个非叶子节点包含 `k-1` 个关键字，`k` 是该节点的子节点个数。
- 根节点至少有 2 个子节点（如果不是叶子节点）。
- 所有叶子节点在同一层。

这意味着：
- 一个节点的关键字数量介于 `⌈m/2⌉-1` 和 `m-1` 之间。
- 一个内部节点最多包含 `m-1` 个关键字和 `m` 个子指针。

## 3. B 树的基本操作

### 3.1 查找

B 树的查找类似于在节点内做一次有序数组二分查找，然后递归进入子树。

因为树高较低，磁盘读写次数少，适合外存索引。

### 3.2 插入

B 树插入过程：
1. 在叶子节点中定位插入位置。
2. 将关键字插入叶节点。
3. 如果节点关键字数超过 `m-1`，则分裂该节点。
4. 将中间关键字上升到父节点，父节点可能继续分裂。

分裂时，一个节点被拆成左右两部分，中间关键字上移，保持各节点关键字数量平衡。

### 3.3 删除

B 树删除过程更复杂，主要包括三种情况：
- 删除叶子节点中的关键字。
- 删除内部节点中的关键字，需要用前驱或后继关键字替换。
- 删除后若节点关键字数不足，则需要借关键字或合并兄弟节点。

删除后通过“节点借取”或“节点合并”，保证每个节点的关键字数量仍然满足下界。

## 4. B+ 树是什么？

B+ 树是 B 树的变种，更适合磁盘索引和范围查询。

其主要区别是：
- 所有关键字都只存储在叶子节点，内部节点只存储索引键。
- 叶子节点按关键字顺序通过指针连接，形成链表。
- 叶子节点包含实际记录或记录指针，内部节点只用于查找路径。

因此，B+ 树的叶子层包含完整的数据，并且支持高效的顺序遍历。

## 5. B 树与 B+ 树的对比

| 特性 | B 树 | B+ 树 |
|------|------|--------|
| 关键字存储位置 | 内部节点和叶子节点 | 仅叶子节点 |
| 叶子节点链接 | 不一定 | 通常按顺序链表连接 |
| 范围查询效率 | 较低 | 更高 |
| 内部节点存储密度 | 较低 | 更高 |
| 非叶子节点大小 | 更大 | 更小 |
| 适用场景 | 一般索引结构 | 数据库、文件系统索引 |

## 6. B+ 树的优势

- 叶子节点链表支持顺序遍历和范围查询。
- 内部节点只存储索引键，节点容量更大，树高更低。
- 更适合磁盘存储，因为每个节点可存储更多索引信息，减少 I/O 次数。
- 叶子节点的数据存储方式一致，便于批量读取。

## 7. 形象理解

从“数形结合”角度看：

- B 树是一棵“扁平的大分支树”，每个节点包含多个关键字，形态比二叉树更宽。
- B+ 树把数据主要放在叶子层，内部节点只负责“分割形状”，使树形更适合顺序访问。

这两种树结构都通过“在节点内存储多个关键字”来降低高度，从而减少磁盘访问。

## 8. 应用场景

- 数据库索引（B+ 树最常见）
- 文件系统目录结构
- 键值存储和外部排序
- 任何需要大规模磁盘索引和范围查询的场景

## 9. 结语

B 树与 B+ 树是面向外存的平衡树结构。它们通过“多路分支”和“节点内多个关键字”来降低树高，并通过分裂与合并操作保持平衡。

其中 B+ 树更适合范围查询和顺序访问，而 B 树适合一般索引存储。
