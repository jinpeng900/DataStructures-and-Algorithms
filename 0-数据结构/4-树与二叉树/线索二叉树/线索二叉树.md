# 线索二叉树

## 1. 什么是线索二叉树

线索二叉树（Threaded Binary Tree）是一种特殊的二叉树结构，它利用二叉树中空的左右指针域存储节点在某种遍历序列下的前驱节点或后继节点信息。

普通二叉树中，节点的左右孩子指针如果为空，则浪费了指针域；线索二叉树将这些空指针重新利用起来，使得树的遍历可以在不使用栈、不使用递归的情况下完成。

## 2. 线索二叉树的基本概念

每个节点有两个指针域：`lchild` 和 `rchild`，还有两个标志域：`ltag` 和 `rtag`。

- `ltag == 0`：`lchild` 指向左孩子
- `ltag == 1`：`lchild` 指向当前节点在遍历序列中的前驱节点
- `rtag == 0`：`rchild` 指向右孩子
- `rtag == 1`：`rchild` 指向当前节点在遍历序列中的后继节点

线程化操作就是把这些原本为空的指针转换成“线索”。

## 3. 为什么要用线索二叉树

- 提高遍历效率，避免递归和栈
- 可以直接找到前驱和后继节点
- 在中序遍历/前序遍历/后序遍历下，支持顺序访问

## 4. 中序线索二叉树

### 4.1 定义

中序线索二叉树是最常见的一种线索二叉树。它按中序遍历顺序建立线索，即每个节点的空左指针指向其中序前驱，每个节点的空右指针指向其中序后继。

### 4.2 线索化过程

使用递归遍历的思路：

1. 递归线索化左子树
2. 处理当前节点：
   - 如果当前节点左孩子为空，则让 `lchild` 指向 `pre`（前一个访问节点），并设置 `ltag = 1`
   - 如果 `pre` 存在且 `pre->rchild` 为空，则让 `pre->rchild` 指向当前节点，并设置 `pre->rtag = 1`
3. 更新 `pre = 当前节点`
4. 递归线索化右子树

通常会维护一个全局或外部变量 `pre`，它始终指向上一个被访问的节点。

### 4.3 遍历操作

#### 中序线索二叉树的遍历

1. 从根节点出发，找到最左的节点（沿着 `ltag==0` 的 `lchild` 走）
2. 访问当前节点
3. 如果 `rtag == 1`，直接沿着 `rchild` 走到当前节点的中序后继并继续访问
4. 如果 `rtag == 0`，将当前节点设为 `rchild`，然后回到步骤 1（寻找其右子树的最左节点）

这样就能做到中序遍历不使用递归或栈。

### 4.4 示例代码思路（中序线索化）

```cpp
void InThreading(Node* T) {
    if (T) {
        InThreading(T->lchild);

        if (T->lchild == nullptr) {
            T->ltag = 1;
            T->lchild = pre;
        }
        if (pre && pre->rchild == nullptr) {
            pre->rtag = 1;
            pre->rchild = T;
        }
        pre = T;

        InThreading(T->rchild);
    }
}
```

### 4.5 中序线索遍历代码思路

```cpp
Node* InOrderNext(Node* p) {
    if (p->rtag == 1) return p->rchild;
    p = p->rchild;
    while (p && p->ltag == 0) p = p->lchild;
    return p;
}

void InOrderTraverse(Node* root) {
    Node* p = root;
    while (p && p->ltag == 0) p = p->lchild;
    while (p) {
        visit(p);
        p = InOrderNext(p);
    }
}
```

## 5. 前序线索二叉树

### 5.1 定义

前序线索二叉树按前序遍历顺序建立线索。对于每个节点：

- 空左指针指向其前序前驱
- 空右指针指向其前序后继

前序线索化后，从根节点开始，通过线索可以按前序顺序访问整棵树。

### 5.2 线索化过程

前序线索化时，访问顺序是“根 -> 左 -> 右”。

1. 处理当前节点：
   - 如果左孩子为空，则将 `lchild` 指向 `pre`，并设置 `ltag = 1`
   - 如果 `pre` 存在且 `pre->rchild` 为空，则令 `pre->rchild = 当前节点`，并设置 `pre->rtag = 1`
2. 更新 `pre = 当前节点`
3. 递归线索化左子树
4. 递归线索化右子树

前序线索化的关键是：先访问当前节点，再访问左子树，最后访问右子树。

### 5.3 遍历操作

前序线索遍历一般按下面方式进行：

1. 从根节点开始访问
2. 如果 `ltag == 0`，直接进入左子树
3. 否则如果 `rtag == 0`，进入右子树
4. 如果 `rtag == 1`，直接跳到前序后继

由于前序线索二叉树的后继关系可能存在于右孩子或线索指针中，因此遍历时要先判断左子树是否存在。

### 5.4 示例代码思路（前序线索化）

```cpp
void PreThreading(Node* T) {
    if (T) {
        if (T->lchild == nullptr) {
            T->ltag = 1;
            T->lchild = pre;
        }
        if (pre && pre->rchild == nullptr) {
            pre->rtag = 1;
            pre->rchild = T;
        }
        pre = T;

        if (T->ltag == 0) PreThreading(T->lchild);
        if (T->rtag == 0) PreThreading(T->rchild);
    }
}
```

### 5.5 前序线索遍历代码思路

```cpp
void PreOrderTraverse(Node* root) {
    Node* p = root;
    while (p) {
        visit(p);
        if (p->ltag == 0) {
            p = p->lchild;
        } else {
            p = p->rchild;
        }
    }
}
```

## 6. 后序线索二叉树

### 6.1 定义

后序线索二叉树按后序遍历顺序建立线索，即每个节点的空左指针指向其后序前驱，空右指针指向其后序后继。

后序线索二叉树最复杂，因为后序遍历是“左 -> 右 -> 根”，根节点的后继通常不是父节点，需要额外处理。

### 6.2 线索化过程

后序线索化同样使用 `pre` 指针，但遍历顺序不同：

1. 递归线索化左子树
2. 递归线索化右子树
3. 处理当前节点：
   - 如果左孩子为空，则将 `lchild` 指向 `pre`，并设置 `ltag = 1`
   - 如果 `pre` 存在且 `pre->rchild` 为空，则将 `pre->rchild = 当前节点`，并设置 `pre->rtag = 1`
4. 更新 `pre = 当前节点`

### 6.3 遍历操作

后序线索遍历相对困难，通常需要借助额外的标志或一个虚拟头结点（dummy head）。

经典方法：

1. 先找到后序第一个节点（从根沿着左孩子和右孩子不断前进，直到叶子节点）
2. 访问当前节点
3. 如果 `rtag == 1`，直接沿 `rchild` 跳到后继节点
4. 如果 `rtag == 0`，说明需要回到父节点或右子树，但由于没有父指针，通常借助头结点或临时变量进行判断

后序线索化后的遍历实现要比中序、前序更复杂，但原理依然是利用线索指针表示后继关系。

### 6.4 示例代码思路（后序线索化）

```cpp
void PostThreading(Node* T) {
    if (T) {
        PostThreading(T->lchild);
        PostThreading(T->rchild);

        if (T->lchild == nullptr) {
            T->ltag = 1;
            T->lchild = pre;
        }
        if (pre && pre->rchild == nullptr) {
            pre->rtag = 1;
            pre->rchild = T;
        }
        pre = T;
    }
}
```

### 6.5 后序线索遍历思路

后序线索遍历常见方法是：

- 使用虚拟头结点，令头结点的左孩子指向根，右指向后序遍历的最后节点
- 对当前节点，如果 `rchild` 是线索，则直接跳到后继
- 如果不是，按照后序顺序寻找下一个节点

这种遍历方法一般需要特殊约定和辅助节点，适合对性能要求不高且追求线索结构完整性的场景。

## 7. 线索二叉树的分类比较

| 类型 | 线索方向 | 遍历顺序 | 优点 | 复杂度 |
|------|----------|----------|------|--------|
| 中序线索二叉树 | 中序前驱、后继 | 左根右 | 最常用，遍历简单 | 中等 |
| 前序线索二叉树 | 前序前驱、后继 | 根左右 | 根节点优先访问 | 简单 |
| 后序线索二叉树 | 后序前驱、后继 | 左右根 | 适用于后序访问 | 最复杂 |

## 8. 使用建议与注意事项

- 中序线索二叉树是最实用的，尤其适合中序遍历频繁的场景。
- 前序线索二叉树适用于需要快速访问根节点和前序关系的场景。
- 后序线索二叉树在实现上最复杂，通常仅在特定需求下使用。
- 线索化后不能再像普通二叉树一样随意插入或删除节点，插入删除需重新维护线索。
- 使用线索指针时，必须明确 `ltag` 和 `rtag` 的含义，避免把普通子指针误当成线索指针。

## 9. 经典结构体定义

```cpp
struct ThreadNode {
    char data;
    ThreadNode* lchild;
    ThreadNode* rchild;
    int ltag;
    int rtag;
};
```

- `ltag == 0` 表示 `lchild` 指向左孩子
- `ltag == 1` 表示 `lchild` 指向前驱
- `rtag == 0` 表示 `rchild` 指向右孩子
- `rtag == 1` 表示 `rchild` 指向后继
